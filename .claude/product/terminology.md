# Terminology

How the product talks about itself, kept distinct from `glossary.md` (which defines domain nouns). This file covers phrasing and naming consistency across UI copy, API responses, and documentation.

## Preferred terms

| Use                    | Not                                             |
| ---------------------- | ----------------------------------------------- |
| _Example: "workspace"_ | _"team," "org," "account" used interchangeably_ |
| _Example: "sign in"_   | _"log in," "login" as a verb_                   |

## Rule

Pick one term per concept and use it everywhere: UI labels, API field names, error messages, and documentation. If a term needs to change, update it in every layer at once rather than letting the frontend and backend drift apart.
